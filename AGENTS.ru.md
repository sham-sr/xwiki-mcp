# xwiki-mcp — сценарий поиска для агента

**[English](AGENTS.md)**

MCP уже настроен. Не угадывайте пути страниц.

## Порядок

`search` → `get_page({ id })` → `get_page_children` / `get_attachments` при необходимости.

Ссылка от пользователя? `resolve_url` → `get_page({ id })`.

## Параметры `search`

| Задача | Вызов |
|--------|--------|
| Раздел по заголовку | `scope:"title"`, `wiki:"…"`, 2–4 ключевых слова |
| Тема по ключевым словам | `wiki:"…"`, короткий `query` |
| Точная фраза из страницы | `query:"\"фраза из вики\""`, `wiki:"…"` |
| Подраздел | добавить `space:"Parent.Child"` |
| Неизвестна область вики | сначала `list_wikis`, затем `wiki` |

**Правила:** из вопроса пользователя — 2–4 слова, не весь абзац (если не в кавычках). `id` из результатов → в `get_page` без изменений. `engine` в ответе — кто вернул результаты, не ваш запрос.

**Пустой ответ:** `_search.suggestions` и `_search.solr_q` → меньше слов или `scope:"title"` → `list_spaces` / `wiki_status`.

## Пример

Вопрос: «Как работает статическая навигация?»

1. `search({ query: "статическая навигация", scope: "title", wiki: "mywiki" })`
2. `search({ query: "руководство навигация", scope: "title", wiki: "mywiki" })`
3. `get_page({ id: "<id из шага 2>" })`

Запись в вики — только по явной просьбе пользователя.
